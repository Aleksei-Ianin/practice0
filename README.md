### 3. Ответы на контрольные вопросы

#### 1. Для чего нужна команда `git init, git clone`?

[git clone — скачивает готовый репозиторий с GitHub/GitLab к тебе на компьютер (со всей историей). После команды появляется папка с проектом.

git init — создает новый пустой репозиторий в текущей папке "с нуля". Появляется скрытая папка .git для отслеживания версий.]

#### 2. Что такое "коммит"?

[Коммит — это "сохранение" текущего состояния проекта.

В коммит входит:

Автор (кто сделал)

Дата (когда)

Сообщение (что изменил)

Снимок файлов (сами изменения)

Создается командой git commit -m "сообщение".]

#### 3. Чем отличаются файлы `.puml` от `.png` в контексте вашей работы?

[.puml — это исходный код диаграммы (текст). Можно редактировать.

.png — это готовая картинка (сгенерирована из кода).

В Git лучше хранить .puml, потому что он текстовый — можно видеть историю изменений и править код. .png можно всегда сгенерировать заново.]


#### 4. Объясните, как в коде Go реализуется связь "класс User использует интерфейс Notifier", показанная на диаграмме.

[В коде:

Интерфейс Notifier — задает правило: "умею отправлять сообщения"

Структуры EmailNotifier и SmsNotifier — выполняют это правило (реализуют Send)

Структура User — имеет поле notifier типа Notifier (интерфейс)

Как работает:
При создании пользователя (NewUser) мы передаем ему нужный notifier (email или SMS).
Когда вызываем user.Notify(), он просто говорит notifier.Send() — а тот сам разбирается, как именно отправлять.

Преимущество: можно легко подменить способ отправки, не меняя код User. Хочешь Telegram-уведомления — просто добавь новый notifier.]
